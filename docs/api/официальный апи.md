# API официального расписания

## Список терминов, глоссарий

хост - rasp.omgtu.ru

эндпоинт - часть URL, после хоста до идентификатора объекта

идентификатор объекта - часть URL после эндпоинта, до параметров, обозначающая тип объекта в системе расписания

параметры - часть URL после идентификатора объектов, отделяется символом `?`

## Краткое введение

Официальный API расписания можно условно разделить на 2 части:

- поисковый API
- календарный API

Интересующее расписание можно получать обратившись в календарному API.

Календарный API может возвращать расписание в двух форматах:

- json
- icalendar

По умолчанию календарный API возвращает расписание в формате json

Для того чтобы получить расписание в формате ics нужно добавить строку `.ics` после идентификатора объекта и перед параметрами

API работает с GET запросами, параметры передаются в URL

## Примеры

### Для группы

`https://rasp.omgtu.ru/api/schedule/group/565?start=2026.02.09&finish=2026.02.15&lng=1`

`565` - идентификатор группы
`start=2026.02.09`, `finish=2026.02.15` - соответственно начало и конец запрашиваемого периода времени
`lng=1` - язык русский (опциональный параметр)

Пример запросы ics версии расписания

`https://rasp.omgtu.ru/api/schedule/group/565.ics?start=2026.02.09&finish=2026.02.15&lng=1`

### Для преподов

`https://rasp.omgtu.ru/api/schedule/person/123?start=2026.02.09&finish=2026.02.15&lng=1`

где `123` - идентификатор препода, остальное то же самое

### Для аудиторий

`https://rasp.omgtu.ru/api/schedule/auditorium/123?start=2026.02.09&finish=2026.02.15&lng=`

где `123` - идентификатор аудитории, остальное то же самое

### Поиск

`https://rasp.omgtu.ru/api/search?term=123&type=person`

где `term=123` - поисковый запрос, `type=person` - преподаватель, `type=auditorium` - аудитория, `type=student` - студент, `type=group` - группа
